## 1. نظرة عامة عن المشروع

**الاسم:** El Prince Bajaj - نظام إدارة مركز خدمة موتوسيكلات  
**المنصة:** Web App كامل (Next.js 15)  
**نوع المشروع:** ERP / POS / CRM لمركز خدمة موتوسيكلات Bajaj  
**الحالة:** قيد التطوير - تم إنجاز 3 مراحل (Phase 1, 2, 3)  
**آخر تحديث:** 26 يونيو 2026  
**النسخة الحالية:** v0.1.0  

### الفريق
- Full Stack: Next.js + TypeScript + Prisma
- 3D: Three.js + React Three Fiber + GSAP
- UI/UX: Tailwind CSS v4 + Framer Motion
- Database: PostgreSQL (Neon)
- Deployment: Vercel

## 2. الفكرة كاملة

### المشكلة
مراكز خدمة الموتوسيكلات في مصر تعاني من:
- إدارة يدوية للمخزون وقطع الغيار
- صعوبة تتبع حجوزات العملاء وأوامر العمل
- عدم وجود نظام نقاط بيع (POS) متكامل
- ضعف التواصل مع العملاء بعد الخدمة
- صعوبة إعداد التقارير المالية
- غياب نظام محاسبي متكامل (دفترية / قيود محاسبية)

### الحل
نظام "El Prince Bajaj" هو حل متكامل يشمل:
1. **موقع شركة (Landing Page)** لعرض الخدمات وجذب العملاء
2. **نظام حجوزات (Booking System)** لحجز مواعيد الصيانة
3. **متجر إلكتروني (Marketplace)** لبيع قطع الغيار
4. **نظام نقاط بيع (POS)** متكامل مع الباركود والفواتير
5. **نظام مخازن (Warehouse)** لإدارة المخزون
6. **نظام محاسبة (Accounting)** بالقيود المزدوجة
7. **نظام CRM** لإدارة العملاء والمركبات
8. **نظام واتساب (WhatsApp Integration)** للتواصل الآلي
9. **نظام صلاحيات (RBAC)** متكامل
10. **لوحة تحكم (Admin Dashboard)** شاملة

### مبدأ عمل النظام

```
العملاء ← موقع الشركة / Booking / Contact
         ↓
    Admin Dashboard ← إدارة كل شىء
         ↓
    POS ← بيع قطع غيار + فواتير
         ↓
    Warehouse ← إدارة المخزون + حركات المخزون
         ↓
    Accounting ← قيود محاسبية + دفترية + فترات محاسبية
         ↓
    WhatsApp ← تواصل آلي + تذكير
```

## 5. قسم الأدمن - شرح كامل

### 5.1 نظرة عامة

قسم الأدمن (`/admin/`) هو قلب النظام. يتم حمله بوساطة Middleware يحمي كل الصفحات:

```
middleware.ts:
  - يتحقق من وجود admin_token في الـ Cookie
  - إذا منتهي، يحاول تجديده (Silent Refresh)
  - يتحقق من الصلاحية (admin/staff/viewer)
  - يمرر user-id, user-role, user-name, tenant-id في الـ Headers
```

### 5.2 نظام المصادقة (Auth)

#### تسجيل الدخول `admin/page.tsx`
- **Client Component** مع Framer Motion
- إدخال Username + Password
- إرسال إلى `POST /api/auth/login/`
- JWT Token (1 ساعة) + Refresh Token (7 أيام)
- تخزين في Cookies (httpOnly)

#### JWT System (`lib/auth.ts`)
- **passwordSchema:** Zod validation (8 أحرف + كبير + صغير + رقم)
- **hashPassword/verifyPassword:** bcryptjs (12 rounds)
- **createToken:** SignJWT مع HS256
- **verifyToken:** jwtVerify
- **createRefreshToken/verifyRefreshToken:** 7 أيام
- **withAuth / withRole / withPermission:** Higher-order functions للمصادقة
- **requireAuth / requireRole / requirePermission:** (Deprecated - للتوافق)

#### نظام الصلاحيات (RBAC) - `lib/permissions.ts`
- **3 أدوار:** admin (كل الصلاحيات), staff (تشغيلية), viewer (قراءة فقط)
- **28 صلاحية** محددة:
  ```
  Dashboard, Products CRUD, Inventory, Import, 
  Invoices CRUD, POS, POS History,
  Customers CRUD, Vehicles CRUD,
  Bookings CRUD, Work Orders CRUD,
  Accounting View, Accounting Manage, Periods Manage,
  Users Manage, Settings, WhatsApp, Upload,
  Reports, Suppliers CRUD
  ```
- صلاحيات قابلة للتخصيص من خلال `Permission` + `RolePermission` tables في DB

### 5.3 القائمة الجانبية (AdminSidebar)

**مكون تفاعلي:**
- قابلة للطي (Collapse/Expand) - تحفظ الحالة في localStorage
- تنقيل تلقائي للموبايل (Drawer + Overlay)

### 5.5 نظام نقاط البيع (POS)

#### `/admin/pos/page.tsx`
**ثلاثة Tabs:** POS | Invoices | Treasury

##### POS Tab:

- **Grid عرض المنتجات** مع صور

  - إدخال يدوي للباركود
